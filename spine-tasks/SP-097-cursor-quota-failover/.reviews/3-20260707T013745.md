### Verdict: REVISE

### Summary

SP-097 adds solid gateway-layer quota classification, circuit-breaker isolation, and `selectCursorQuotaFailover` with `cursor_quota_exhausted` telemetry in `gateway-dispatch.ts`, backed by focused unit tests. However, the build gate fails intermittently due to a non-deterministic economical-failover assertion, and the primary dogfood usage-limit message path does not trigger end-to-end failover because `shouldFailoverOnProviderError` is exported but never wired into the delegation layer.

### Build / typecheck

Command: `npm run typecheck && npm run lint && npm test`

- `typecheck` and `lint` passed.
- `npm test` failed on first run (1/5 full-suite reruns also failed):

```
FAIL tests/unit/gateway-dispatch.test.ts > GatewayDispatch > cursor quota exhaustion failover (SP-097) > fails over to economical API model when cursor/auto unavailable
AssertionError: expected 'gpt-4o-mini' to be 'gemini-flash-lite-latest'
```

`selectCursorQuotaFailover` uses `weightedSelect`, which probabilistically picks among economical candidates; the test asserts a single fixed model ID.

Coverage (`npm run coverage:check`) passed when tests passed; `gateway-dispatch.ts` line coverage is 95.88% (≥77% threshold).

### Blocking issues

1. **`tests/unit/gateway-dispatch.test.ts:321-356`** — The economical-failover test asserts `gemini-flash-lite-latest` but `weightedSelect` randomly returns either economical candidate (`gpt-4o-mini` observed); this makes `npm test` / `verify:ci` flaky.
   - **Missing test fix:** Assert the result is one of the economical candidates and has `reason_code: 'cursor_quota_exhausted'`, or stub/mock `weightedSelect` / use a single economical candidate.

2. **`.pi/extensions/smart-router/route-and-delegate.ts:206-240`** — Failover is gated on `isInfraAssistantError` only; plain-text usage-limit messages (dogfood #70: "You've hit your usage limit…") do not parse as infra errors, so `selectFailover` is never called even after `recordOutcome` marks quota exhaustion.
   - **Missing integration:** Wire `shouldFailoverOnProviderError` (exported from `gateway-dispatch.ts:115-128`) into the delegation error path, or add an extension-layer test proving message-based quota errors retry to `cursor/auto`/economical model.

### Suggestions (non-blocking)

- Consider adding `cursor_quota_exhausted` to any reason-code allowlists or telemetry docs if the project maintains explicit enumerations.
- `isCursorAutoModel` treats all `cursor/*` IDs as auto models (`gateway-dispatch.ts:106-109`), which may overlap with `isCursorSubscriptionModel` for subscription frontier models — worth a follow-up if misclassification appears in production fleets.
